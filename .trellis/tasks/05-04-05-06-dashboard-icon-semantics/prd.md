# Dashboard icon semantics

## Goal

Fix the 5 dashboard shortcut icons whose current `UIIcon` mapping is
either semantically wrong or visually duplicated, by adding new
`UIIcon` values + matching 32/64-px assets and re-wiring
`DashboardShortcutStore::kDefinitions`.

## What I already know

- Current mapping in `src/DashboardShortcutStore.cpp:14-35`. 11 shortcuts,
  five misalignments:

| # | Shortcut | Current icon | Problem |
|---|---|---|---|
| 1 | WeatherClock (天氣與時間) | `Recent` (clock) | Missing 天氣 half — original user complaint |
| 2 | Today (今日) | `Recent` (clock) | Duplicates WeatherClock |
| 3 | RecentReading (繼續閱讀) | `Recent` (clock) | Duplicates the same clock |
| 4 | ArcadeHub (遊戲廳) | `Hotspot` (signal waves) | Wrong: games != hotspot |
| 5 | DesktopHub (桌面) | `Folder` | Duplicates FileBrowser |

- `UIIcon` enum in `src/components/themes/BaseTheme.h:73`:
  `{Folder, Text, Image, Book, File, Recent, Settings, Transfer, Library,
   Wifi, Hotspot}`. No weather, calendar, joystick, bookmark, or grid
  values.
- Each icon has 32×32 (`*Icon`) and 64×64 (`*64Icon`) variants under
  `src/components/icons/`. Existing files: book / cog / cover / file24 /
  folder / hotspot / image24 / library / recent / settings(2) / text24 /
  transfer / wifi. 32-px assets ≈ 880 bytes, 64-px ≈ 3.3 KB. All 1-bit,
  top-down, MSB-first, byte-packed.
- DashboardShortcutDefinition couples `DashboardShortcutId → UIIcon` at
  construction time; dashboard renders 64×64 in iOS-grid mode and 32×32
  in list mode.
- iOS-grid 64×64 icon helper: `iconBitmap64ForName()` in
  `src/activities/home/DashboardActivity.cpp`. List mode 32×32 helper is
  the LyraTheme `drawList(rowIcon=...)` path which calls another mapper.

## Assumptions (temporary)

- 5 new `UIIcon` values + 5 × 2 = 10 new asset files (.h with byte arrays).
- Asset format follows existing 1-bit `uint8_t[]` convention. No runtime
  BMP loading.
- Asset generation is automated from monochrome SVG/PIL, not hand-drawn,
  to keep iteration cheap.
- The 6 icons already deemed correct (ReadingHub→Library, StudyHub→Book,
  ImportSync→Transfer, FileBrowser→Folder, Settings→Settings,
  StudyToday→Book) are out of scope — no edits.

## Decision (ADR-lite — Q1 scope)

**Context**: 5 of 11 dashboard shortcut icons are misaligned;
WeatherClock alone is the user's most visible complaint but the same
clock-face icon is also used by Today and RecentReading.

**Decision**: Fix all 5 in this task (Option 2 from the brainstorm).
Smallest scope (Option 1) leaves three duplicate-clock cells. Largest
(Option 3 — full 11-icon redesign) is out of scope for now.

**Consequences**: ~5 new icon BMPs to author, 1 enum mapping change in
`DashboardShortcutStore::kDefinitions`, 1 enum extension in
`BaseTheme.h`, 2 `iconBitmapForName`/`iconBitmap64ForName` updates.
~150 lines of generated byte data + ~40 lines of plumbing.

## Glyph picks (Bootstrap Icons, MIT)

Source: <https://icons.getbootstrap.com>. Single ~110 KB TTF, MIT, 2000+
glyphs. Outline style matches the existing 1-bit `book.h` look.

| Shortcut | Bootstrap Icon | Reasoning |
|---|---|---|
| WeatherClock 天氣與時間 | `bi-cloud-sun` | Cloud + sun; "weather" reads clearly, time half deferred to subtitle |
| Today 今日 | `bi-calendar3` | Plain calendar grid |
| RecentReading 繼續閱讀 | `bi-bookmark` | Bookmark = "continue reading" |
| ArcadeHub 遊戲廳 | `bi-controller` | Game controller silhouette |
| DesktopHub 桌面 | `bi-grid-3x3-gap` | 3×3 dots; matches dashboard's iOS-grid aesthetic |

## Requirements (evolving)

- Extend `UIIcon` enum in `src/components/themes/BaseTheme.h` with 5
  new values (provisional names):
  `Weather`, `Calendar`, `Bookmark`, `Joystick`, `AppGrid`.
- Add 10 new asset files under `src/components/icons/`:
  `weather.h` + `weather64.h`, `calendar.h` + `calendar64.h`,
  `bookmark.h` + `bookmark64.h`, `joystick.h` + `joystick64.h`,
  `appgrid.h` + `appgrid64.h`. Each follows the `BookIcon` /
  `Book64Icon` byte-array convention with `// size: WxH` header comment.
- Update `iconBitmap64ForName(UIIcon)` in
  `src/activities/home/DashboardActivity.cpp` and the 32-px mapper used
  by LyraTheme list rows to handle the new enum values.
- Update `DashboardShortcutStore::kDefinitions` to remap:
  - WeatherClock → `UIIcon::Weather`
  - Today → `UIIcon::Calendar`
  - RecentReading → `UIIcon::Bookmark`
  - ArcadeHub → `UIIcon::Joystick`
  - DesktopHub → `UIIcon::AppGrid`
- Build clean (`pio run -e mofei`); flash + take screenshot evidence.

## Technical Notes

- Existing icon style: 1-bit, top-down, MSB-first, byte-packed rows.
- See `book.h` (32×32) and `book64.h` (64×64) as canonical examples.
- The dashboard cell box at iOS-grid renders the icon centered above
  label + subtitle; new assets should leave a 2-3 px transparent border.
- Generation tool plan (if Q1 = option 1): a one-shot
  `scripts/scratch/build_dashboard_icons.py` reads source PNG/SVG from
  the same dir, applies threshold, writes the `.h` files. Tool is
  scratch-only (gitignored) and not part of the build.
